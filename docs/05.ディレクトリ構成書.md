## 3. バックエンド（backend/）

```
backend/
├── Dockerfile                  # Python 3.12 slim ベースイメージ
├── requirements.txt            # Python パッケージ一覧
├── alembic.ini                 # Alembic設定
├── alembic/
│   ├── env.py                  # マイグレーション実行設定（async対応）
│   ├── script.py.mako          # マイグレーションスクリプトテンプレート
│   └── versions/
│       └── 001_initial.py      # 初期マイグレーション（3テーブル + トリガー）
└── app/
    ├── __init__.py
    ├── main.py                 # FastAPI アプリケーション定義・ルーター登録・CORS設定
    ├── config.py               # 環境変数ベースの設定クラス
    ├── database.py             # SQLAlchemy async engine/session
    ├── dependencies.py         # JWT認証依存関数（get_current_user）
    ├── models/
    │   ├── __init__.py         # モデルの re-export
    │   ├── user.py             # User モデル + DeclarativeBase
    │   ├── transaction_rule.py # TransactionRule モデル
    │   └── transaction.py      # Transaction モデル
    ├── schemas/
    │   ├── __init__.py
    │   ├── auth.py             # 認証系リクエスト/レスポンス
    │   ├── rule.py             # ルール系リクエスト/レスポンス
    │   ├── transaction.py      # トランザクション系リクエスト/レスポンス
    │   └── dashboard.py        # ダッシュボード系レスポンス
    ├── services/
    │   ├── __init__.py
    │   ├── auth.py             # パスワードハッシュ・JWT生成/検証・ユーザーCRUD
    │   ├── rule.py             # ルールCRUD
    │   ├── transaction.py      # トランザクションCRUD・ステータス更新・ダッシュボード集計
    │   └── batch.py            # 月次データ生成・繰越処理
    └── routers/
        ├── __init__.py
        ├── auth.py             # POST /api/v1/auth/*
        ├── rules.py            # GET/POST/PUT/DELETE /api/v1/rules/*
        ├── transactions.py     # GET/POST/PUT/PATCH/DELETE /api/v1/transactions/*
        ├── dashboard.py        # GET /api/v1/dashboard/summary
        └── batch.py            # POST /api/v1/batch/*
```

### レイヤー構成

| レイヤー | ディレクトリ | 責務 |
|---------|------------|------|
| ルーティング | routers/ | HTTPリクエストの受付・レスポンス返却 |
| スキーマ | schemas/ | リクエスト/レスポンスのバリデーション・シリアライズ |
| サービス | services/ | ビジネスロジック・DB操作 |
| モデル | models/ | SQLAlchemy ORM モデル定義 |
| インフラ | database.py, config.py | DB接続・設定管理 |

## 5. Docker構成

| サービス | イメージ | ポート | 説明 |
|---------|---------|--------|------|
| db | postgres:16-alpine | 5432 | PostgreSQL データベース |
| backend | Python 3.12 (カスタム) | 8000 | FastAPI アプリケーション |
| frontend | Node 20 (カスタム) | 3000 | Next.js アプリケーション |

### 依存関係

```
frontend → backend → db
```

- `db`: ヘルスチェック付き。PostgreSQL が ready になるまで backend は待機
- `backend`: 起動時に Alembic マイグレーションを自動実行
- `frontend`: backend 起動後に起動
